# VMC design QA — 9 июля 2026

## Основание

- ТЗ: `Корректировки сайт.docx`, 12 пунктов.
- Визуальный референс: https://robot.automacon.ru/.
- Контентные референсы: https://www.aseng.ru/ase, https://www.aseng.ru/services1, https://www.aseng.ru/portfolioindustry.
- Контент и реквизиты VMC сверялись с https://vmc34.ru/.

## Результат

- Все 12 пунктов ТЗ представлены в коде.
- Hero использует статичное локальное изображение без видео и фоновой анимации.
- Каталог содержит шесть решений с адаптированными под VMC описаниями.
- Клиентская полоса содержит Технониколь, Черкизово и Русскую медную компанию.
- Проекты разделены по трем отраслям; для трех кейсов есть отдельные страницы с прямыми ссылками.
- Контактный блок содержит реквизиты VMC, поля из референса, два отдельных обязательных согласия и страницу политики.

## Проверено

- Codex in-app Browser: 1440×900, 768×900, 375×812 и 320×720.
- На проверенных размерах `scrollWidth` равен ширине viewport; горизонтального переполнения нет.
- Hero-изображение загружается как локальный WebP без видео-цикла.
- Шесть HTML-страниц разобраны `HTMLParser`; все локальные страницы, якоря и assets разрешаются.
- CSS: 251 открывающая и 251 закрывающая скобка; `transition: all` и `will-change: all` отсутствуют.
- Локальный HTTP возвращает `200` для всех страниц, CSS и hero-изображения.

## Ограничения публикации

- Форма намеренно использует `mailto:`: после нажатия пользователь подтверждает отправку в своей почтовой программе.
- GitHub push не публикует сайт автоматически. По `README.md` обновление `vmc34.ru` выполняется отдельной загрузкой файлов в Timeweb `public_html`.
- Политику и согласие перед production-публикацией стоит подтвердить у юриста и у ответственного за обработку персональных данных.
